# Feature Specification: 将棋ゲーム (Shogi Game)

**Feature Branch**: `001-shogi-game`

**Created**: 2026-05-23

**Status**: Draft

**Input**: User description: "将棋ゲームを作って"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 対局を開始して駒を動かす (Priority: P1)

プレイヤーが新しい将棋の対局を開始し、初期局面から交互に駒を動かして対局を進められる。

**Why this priority**: 将棋ゲームの根幹となる機能。盤面表示・駒の移動・手番管理がなければ
他のどのユーザーストーリーも成立しない。

**Independent Test**: 対局を開始し、先手が初手を指し、後手が応手を指せれば、このストーリーは
独立して検証可能。

**Acceptance Scenarios**:

1. **Given** ゲームが起動している状態、**When** 「対局開始」を選択する、**Then** 9×9の将棋盤が
   表示され、先手・後手の駒が初期配置に並び、先手の手番であることが示される
2. **Given** 先手の手番、**When** 自分の駒を選択する、**Then** 選択された駒が強調表示され、
   合法な移動先がハイライトされる
3. **Given** 駒が選択されている状態、**When** ハイライトされた移動先を選択する、**Then**
   駒が移動し、手番が後手に交代する
4. **Given** 相手の駒に移動した場合、**When** 駒を取る、**Then** 取った駒が自分の持ち駒として
   追加される
5. **Given** 自分の手番でない状態、**When** 操作しようとする、**Then** 操作が受け付けられない

---

### User Story 2 - 駒を成る (Priority: P2)

相手陣（最奥3段）に入った駒を「成り駒」に変えることができる。

**Why this priority**: 将棋の基本ルールであり、P1の対局体験が完成した後に独立して追加できる。

**Independent Test**: 歩兵を相手陣3段目まで進め、成り/不成を選択できれば独立して検証可能。

**Acceptance Scenarios**:

1. **Given** 自駒が相手陣に移動した場合、**When** 移動が完了する、**Then**
   「成る」「不成」の選択肢が表示される（成れる駒の場合のみ）
2. **Given** 「成る」を選択した場合、**When** 確定する、**Then**
   駒が成り駒の見た目に変わり（例：歩→と金）、以降は成り駒として動く
3. **Given** 「不成」を選択した場合、**When** 確定する、**Then** 駒はそのままの状態で移動完了
4. **Given** 行き所のない駒（例：歩を最奥の段）を移動させようとする場合、**When** 移動先を選択する、
   **Then** 必ず成る（強制成り）

---

### User Story 3 - 持ち駒を打つ (Priority: P3)

取得した持ち駒を盤上の空きマスに打つことができる。

**Why this priority**: 将棋独自の「打ち」ルールであり、P1・P2の後に独立して追加できる。

**Independent Test**: 相手から歩を取得し、その歩を任意の合法マスに打てれば独立して検証可能。

**Acceptance Scenarios**:

1. **Given** 持ち駒がある状態、**When** 持ち駒エリアの駒を選択する、**Then**
   打てる合法なマスがハイライトされる
2. **Given** 打つ位置を選択する、**When** 確定する、**Then** 駒が盤上に配置され、持ち駒から消える
3. **Given** 歩の持ち駒がある状態で、既に自分の歩がある筋に打とうとする、**When** 打とうとする、
   **Then** そのマスはハイライトされず打てない（二歩禁止）
4. **Given** 駒を打った場合、**When** 確定する、**Then** 打ち歩詰めになる場合は禁じ手として弾かれる

---

### User Story 4 - AI（コンピュータ）と対局する (Priority: P4)

プレイヤーがコンピュータ相手に将棋を対局できる。難易度を選んでAIと一人でも遊べる。

**Why this priority**: 一人でも遊べるようにする重要な機能。P1〜P3の対局基盤が完成してから追加できる。

**Independent Test**: AIモードを選択し、AIが自動で指し手を返してくれれば独立して検証可能。

**Acceptance Scenarios**:

1. **Given** ゲームが起動している状態、**When** 「AI対戦」を選択する、**Then**
   難易度選択画面が表示される（最低2段階）
2. **Given** 難易度を選択した場合、**When** 対局が開始される、**Then**
   プレイヤーが一手指すと、AIが自動で応手を指す
3. **Given** AIの手番、**When** AIが考慮中、**Then** 思考中であることが視覚的に示される
4. **Given** AIモードで対局中、**When** プレイヤーが初級を選ぶ、**Then**
   AIは将棋を始めたばかりのプレイヤーが楽しめる強さで対局する

---

### User Story 5 - 勝敗判定と対局終了 (Priority: P5)

王手・詰みが正確に検出され、対局の勝敗が確定する。また、投了によっても対局を終了できる。

**Why this priority**: ゲームの終了条件として必要だが、P1〜P4が揃ってから独立して追加できる。

**Independent Test**: 詰みの局面を手動で再現し、ゲーム終了画面が正しく表示されれば検証可能。

**Acceptance Scenarios**:

1. **Given** ある手を指した後に相手玉が王手になる場合、**When** その手が完了する、**Then**
   「王手」の通知が表示される
2. **Given** 王手がかかっている状態で相手が詰みの状態、**When** 手番が来る、**Then**
   詰みが検出され、指した側の勝利として対局終了画面が表示される
3. **Given** 対局中、**When** 「投了」を選択する、**Then** 確認ダイアログが表示され、
   確定すると相手の勝利として対局が終了する
4. **Given** 対局終了後、**When** 結果画面を確認する、**Then** 棋譜（全手の履歴）が表示される

---

### Edge Cases

- 行き所のない駒（桂馬を最奥2段・歩を最奥1段など）への移動は禁じ手として拒否する
- 打ち歩詰めは禁じ手として拒否する
- 自殺手（自玉を王手にさらす手）は禁じ手として拒否する
- 千日手（同じ局面が4回繰り返される）は引き分けとして処理する
- 駒を選択後、移動先を選ばずにキャンセル（選択解除）できる

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: システムは9×9の将棋盤と各駒を正確な初期配置で表示しなければならない
- **FR-002**: システムは全ての駒種の合法手をルールに従って正確に計算しなければならない
  （歩・香・桂・銀・金・角・飛・玉、および各成り駒）
- **FR-003**: システムは先手・後手の手番を管理し、手番外の操作を拒否しなければならない
- **FR-004**: システムは取った駒を攻撃側プレイヤーの持ち駒として登録しなければならない
- **FR-005**: システムは駒が相手陣3段に入った際、成りが可能な場合は成り/不成の選択を
  提示しなければならない
- **FR-006**: システムは行き所のない駒への移動を強制成りまたは禁じ手として処理しなければならない
- **FR-007**: システムは持ち駒の打ち込みにおいて二歩・打ち歩詰め・行き所のない位置への打ちを
  禁じ手として拒否しなければならない
- **FR-008**: システムは自殺手を禁じ手として拒否しなければならない
- **FR-009**: システムは王手状態を検知してプレイヤーに通知しなければならない
- **FR-010**: システムは詰みを正確に検知し、勝敗を確定しなければならない
- **FR-011**: システムは千日手を検知し、引き分けとして処理しなければならない
- **FR-012**: プレイヤーはいつでも投了を選択して対局を終了できなければならない
- **FR-013**: システムは全対局の棋譜（各手の記録）を対局終了まで保持しなければならない
- **FR-014**: システムは「人間対人間」と「人間対コンピュータ（AI）」の両対局モードを
  提供しなければならない
- **FR-015**: 対局開始時にプレイヤーは対局モードを選択できなければならない
- **FR-016**: AIモードでは、プレイヤーは少なくとも2段階以上の難易度（例：初級・上級）を
  選択できなければならない

### Key Entities

- **盤面 (Board)**: 9×9のマス目。各マスは空または駒を保持する
- **駒 (Piece)**: 種類（歩・香・桂・銀・金・角・飛・玉）、所有者（先手/後手）、成り状態を持つ
- **プレイヤー (Player)**: 先手または後手。持ち駒のリストを保持する
- **対局 (Game)**: 盤面・両プレイヤー・現在の手番・棋譜・勝敗状態を管理する
- **手 (Move)**: 移動元と移動先、または打ち駒と打つ位置、成り選択の情報を含む
- **棋譜 (Game Record)**: 対局で指された全ての手を時系列で記録したリスト

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: プレイヤーは起動から1分以内に新しい対局を開始できる
- **SC-002**: システムは全ての合法手を正確に提示し、禁じ手を100%の精度で拒否する
- **SC-003**: 詰みの局面が正確に検出される（誤検知・見逃しゼロ）
- **SC-004**: 駒の選択から移動確定までの操作が3ステップ以内で完了する
- **SC-005**: 対局の棋譜が対局開始から終了まで全ての手を漏れなく記録する
- **SC-006**: 将棋のルールを知るプレイヤーが、説明書なしで対局を開始・完了できる
- **SC-007**: AI対戦モードで、初級AIはプレイヤーが操作してから5秒以内に応手を返す

## Assumptions

- 標準的な平手将棋のルール（本将棋）を採用する（駒落ち・変則将棋はv1対象外）
- 時間制御（持ち時間・秒読み・フィッシャークロック）はv1の対象外とする
- オンライン対戦（ネットワーク越しの通信対局）はv1の対象外とする
- 対局の保存・再開機能はv1の対象外とする
- 棋譜の入出力（KIF/CSA形式等）はv1の対象外とする
- 対局のリプレイ機能はv1の対象外とする
- AI難易度は「初級」「上級」の2段階をv1で提供する（それ以上の段階はv1対象外）
- AI同士の自動対局（AI vs AI）はv1の対象外とする
