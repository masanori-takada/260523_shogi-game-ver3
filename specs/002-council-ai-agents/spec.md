# Feature Specification: 脳内三軍師エージェント（合議制AI）

**Feature Branch**: `002-council-ai-agents`

**Created**: 2026-05-24

**Status**: Draft

**Input**: 役割の異なる3つのサブエージェント（猛将・智将・審判）と親エージェント（総大将）による合議制将棋AI。Strands Agents TypeScript SDKを使って実装。

---

## User Scenarios & Testing *(mandatory)*

### User Story 1 — AI対戦で合議制の応手を体験する (Priority: P1)

プレイヤーがAI対戦モードを選択すると、AIの手番に「三軍師の審議」パネルが表示される。猛将・智将・審判それぞれの意見と、総大将の最終判断が可視化されながらAIが次の一手を指す。

**Why this priority**: 本機能の核心価値。3つのエージェントが合議して手を決める様子をプレイヤーが目撃できることが、従来の1エンジンAIとの差別化ポイント。

**Independent Test**: AI対戦モードを開始し、プレイヤーが一手指した後に「三軍師の審議」パネルが表示され、猛将・智将・審判の意見が出揃い、総大将が最終手を選択してAIが着手すれば、US1 は独立して検証完了。

**Acceptance Scenarios**:

1. **Given** AI対戦モードを開始しプレイヤーが初手を指した状態、**When** AIの思考が始まる、**Then** 「三軍師の審議」パネルに猛将・智将・審判それぞれの候補手とスコアが表示され、総大将の最終採用手が強調表示される
2. **Given** 三軍師の審議中、**When** 審判が「危険度高（DANGER）」と判定した場合、**Then** パネルに「⚠️ 危険度高：智将の守り手を採用」と表示され、猛将の手ではなく智将の提案手が採用される
3. **Given** 三軍師の審議中、**When** 猛将が相手玉の3手詰みを発見した場合、**Then** パネルに「✅ 詰み発見：猛将の攻め手を最優先」と表示され、猛将の提案手が採用される
4. **Given** 上記以外の通常局面、**When** 審議が完了する、**Then** 審判の形勢スコアに基づいた重み付けで猛将・智将の意見が統合され、総大将が最終手を決定する

---

### User Story 2 — 意思決定ルールの働きを観察する (Priority: P2)

プレイヤーは対局中、総大将がどのルールを適用して最終手を選んだかの「理由」をパネルで確認できる。これにより、合議制AIの思考プロセスへの理解と信頼が生まれる。

**Why this priority**: パネルに「どの手がなぜ採用されたか」が表示されることで、プレイヤーはAIを学習・信頼でき、リプレイ性が向上する。

**Independent Test**: 危険局面・詰み発見・通常局面の3パターンでAIを動かし、パネルの「採用理由」が各ルールを正しく反映していることを確認すれば US2 は独立して検証完了。

**Acceptance Scenarios**:

1. **Given** 総大将が意思決定ルールを適用した後、**When** パネルを確認する、**Then** 採用された意思決定ルール（RULE-1: 危険回避 / RULE-2: 詰み優先 / RULE-3: 重み付け統合）がテキストで表示される
2. **Given** 過去の手履歴を見る、**When** 三軍師審議ログを確認する、**Then** 各手番での三軍師の意見と総大将の判断が記録として残っている

---

### User Story 3 — 難易度選択に「エージェントAI」が加わる (Priority: P3)

AI対戦モードの難易度選択画面に「初級」「上級」に加えて「エージェントAI」が第3の選択肢として表示される。「初級」「上級」は従来どおり単一のMinimaxエンジンで動作し、「エージェントAI」を選んだときのみ三軍師合議制エンジンが起動し、「三軍師の審議」パネルが表示される。

**Why this priority**: プレイヤーが「いつもの将棋AI」と「新しい合議制AI」を明確に選び分けられることで、合議制AIの特別感が際立つ。また既存の初級・上級ユーザーへのデグレを防ぐ。

**Independent Test**: 難易度選択画面で「エージェントAI」を選んで対局を開始したときのみ三軍師パネルが表示され、「初級」「上級」では従来どおりパネルが非表示のまま動作すれば US3 は独立して検証完了。

**Acceptance Scenarios**:

1. **Given** AI対戦の難易度選択画面、**When** 選択肢を確認する、**Then**「初級」「上級」「エージェントAI」の3択が表示される
2. **Given** 「初級」または「上級」を選択した場合、**When** AI対戦を開始する、**Then** 三軍師の審議パネルは表示されず、従来のMinimaxエンジンがそのまま動作する
3. **Given** 「エージェントAI」を選択した場合、**When** AI対戦を開始しプレイヤーが一手指す、**Then** 三軍師の審議パネルが表示され、合議制エンジンがAIの手を決定する
4. **Given** 「エージェントAI」モードで対局中、**When** 難易度設定を「初級」に変更して再対局する、**Then** 三軍師パネルが非表示に切り替わり、Minimaxエンジンに戻る

---

### Edge Cases

- 合法手が1手しかない局面では、三軍師全員が同じ手を提案し総大将はその手を採用する（審議ショートカット）
- 猛将・智将が同一手を提案し、審判が「DANGER」でない場合は、審議スコアに関わらずその手を採用する
- 猛将が詰みを発見したが局面を精査すると「誤詰み（自玉が危ない）」だった場合、審判がDAN GERを上書きして智将の手を優先する
- Strands エージェントのタイムアウト（10秒超過）が発生した場合は、既存のMinimaxエンジンをフォールバックとして使用する

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: システムは、AI対戦モードの手番計算に猛将・智将・審判・総大将の4エージェント構成の合議制エンジンを使用しなければならない
- **FR-002**: 猛将エージェントは、相手玉への詰み・駒得を最優先する攻撃的評価関数を使って候補手を提案しなければならない
- **FR-003**: 智将エージェントは、自玉の安全度・詰めろ解除・相手の罠回避を最優先する防御的評価関数を使って候補手を提案しなければならない
- **FR-004**: 審判エージェントは、現在局面の有利不利・将棋格言（金銀3枚の囲い・遠見の角など）への適合度から「危険度（SAFE / CAUTION / DANGER）」と「形勢スコア」を算出しなければならない
- **FR-005**: 総大将エージェントは、以下の優先順位で意思決定を行わなければならない：
  - RULE-1: 審判が「DANGER」→ 智将の提案手を強制採用
  - RULE-2: 猛将が3手詰み以内を発見 → 猛将の提案手を強制採用
  - RULE-3: それ以外 → 審判の形勢スコアで猛将・智将の提案を重み付け選択
- **FR-006**: 対局中の三軍師の審議結果（各エージェントの候補手・スコア・採用理由）を「三軍師の審議」パネルに表示しなければならない
- **FR-007**: システムは、エージェント処理がタイムアウト（10秒超過）した場合、既存のMinimaxエンジンにフォールバックしなければならない
- **FR-008**: AI対戦モードの難易度選択肢は「初級」「上級」「エージェントAI」の3種類でなければならない。「初級」（探索深さ2）・「上級」（探索深さ4）は従来のMinimaxエンジンで動作し、「エージェントAI」のみが三軍師合議制エンジンを使用しなければならない
- **FR-009**: 合議制エンジンは、既存の人間対人間モードには影響を与えてはならない

### Key Entities

- **CouncilDecision**: 一手番の審議結果。猛将の候補手・智将の候補手・審判の評価・総大将の採用手・採用理由を含む
- **SubAgentProposal**: サブエージェント1体の提案。候補手（Move）・評価スコア・役割（ATTACKER / DEFENDER）を含む
- **StrategistAssessment**: 審判の評価結果。危険度（SAFE / CAUTION / DANGER）・形勢スコア・格言違反フラグを含む
- **CommanderRuleApplied**: 総大将が適用した意思決定ルール（RULE-1 / RULE-2 / RULE-3）

---

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 「エージェントAI」モードで対局を開始してから最初の三軍師審議結果が表示されるまでの時間が15秒以内。「初級」「上級」モードでは従来どおり5秒以内の応答を維持する
- **SC-002**: RULE-1（DANGER時に智将手強制採用）とRULE-2（3手詰み発見時に猛将手優先）が、対応局面で100%正しく適用される
- **SC-003**: タイムアウトフォールバックが発生した場合でも、プレイヤーの操作が中断されることなく対局が継続できる
- **SC-004**: 既存の人間対人間モードのテスト（37件）がすべてパスし続ける（デグレなし）
- **SC-005**: 三軍師パネルに、各手番で猛将・智将・審判・総大将の4者全員の出力が必ず表示される

---

## Assumptions

- 各サブエージェント（猛将・智将・審判）は、ClaudeのLLMではなく既存の数値評価エンジン（Minimax・格言チェック）を内部で呼び出す関数として実装する（Strands AgentsのToolとして定義）
- 総大将エージェントのみがStrandsの意思決定ループを実行し、3つのサブエージェントをToolとして呼び出す構成とする
- Anthropic APIキーが環境変数（`ANTHROPIC_API_KEY`）に設定されていることを前提とする
- 難易度選択肢は「初級」「上級」「エージェントAI」の3種類とする。「エージェントAI」のみが三軍師合議制エンジンを使用し、「初級」「上級」は既存のMinimaxエンジンをそのまま使用する
- 既存の `src/ai/` ディレクトリ内の評価関数・Minimaxエンジンを合議制システムから再利用する
- モバイル対応（三軍師パネルのレスポンシブ表示）はv1スコープ外とし、デスクトップブラウザのみを対象とする
- 棋譜エクスポートには合議制の審議ログは含めない（対局結果のみ）
